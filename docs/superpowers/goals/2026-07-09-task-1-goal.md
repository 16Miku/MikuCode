## Executable `/goal` body

实现 MikuCode B 版 Task 1：项目骨架 + CLI 入口 + .miku 初始化。严格按 docs/superpowers/plans/2026-07-07-mikucode-implementation.md 的 Task 1 执行，不得提前实现 Task 2+ 功能。

### 1. 目标（What / Why）

在仓库 A:\study\AI\LLM\Fable-5 中，从“仅有设计/计划文档”的状态，落地可安装、可测试的 Python 包脚手架：

- 包名：`mikucode`
- CLI 入口：`miku`（映射到 `mikucode.cli.main:app`）
- 能力：`miku init` 创建 `.miku/` 项目目录；`miku chat` 进入最小 REPL（可 `/exit`）；无参 one-shot 任务仅打印占位消息
- 原则：CLI 层只做用户交互与初始化，**不**包含 agent 决策、工具执行、模型调用、patch、trace 逻辑

本 Task 是后续所有 runtime 能力的地基：路径约定、配置对象、CLI 命令形状、测试与 lint 流水线必须先稳定。

### 2. 范围（In Scope）

仅创建/修改以下文件（与计划一致）：

- `pyproject.toml`
- `.gitignore`
- `README.md`（最小可读即可：项目一句话 + quickstart，不必写完整 architecture）
- `src/mikucode/__init__.py`
- `src/mikucode/config.py`
- `src/mikucode/cli/main.py`
- `src/mikucode/cli/repl.py`
- `tests/test_cli_init.py`

接口契约（必须实现且可被后续 Task import）：

- `MikuConfig`（frozen dataclass）：`project_root`, `miku_dir`, 默认 `permission_mode="auto-safe"`, `default_test_command="uv run pytest"`
- `ensure_miku_dir(project_root: Path) -> Path`
- `load_config(project_root: Path) -> MikuConfig`
- Typer `app` 暴露：`init`、`chat`、以及无参时的 one-shot 回调

### 3. 明确非目标（Out of Scope / 禁止事项）

禁止在本 Task 中实现或“顺手加”：

- AgentRuntime / ModelProvider / ToolRegistry / PatchEngine / TraceRecorder
- 真实模型调用、权限策略、shell、verification、memory 读写逻辑（除初始化写默认文件外）
- MCP、Web、Docker、IDE 集成
- 改动设计文档或实现计划正文（除非用户明确要求更新 checkbox；默认不改 plan 文件）
- 引入计划外依赖；不升级到与计划冲突的栈
- 大规模 README / architecture 文档（留给 Task 9）

若发现需要额外文件（如 `cli/__init__.py`、`src/mikucode/cli/__init__.py`）仅为使包可导入：允许，但保持空/极简，不塞业务逻辑。

### 4. 环境与工程约束

- Python `>=3.11`，**所有** Python 依赖安装、运行、测试、lint 必须用 `uv`：
  - `uv sync`
  - `uv run pytest ...`
  - `uv run ruff check ...`
  - `uv run miku ...`（可选手动 smoke）
- 源码布局：`src/` layout；`pythonpath = ["src"]` 在 pytest 配置中生效
- 依赖（与计划对齐）：
  - runtime：`typer`, `rich`, `pydantic`, `httpx`（本 Task 可不使用 pydantic/httpx，但仍写入 pyproject 以保持后续 Task 一致）
  - dev：`pytest`, `ruff`
- Windows（当前环境）注意：
  - 路径用 `pathlib.Path`，不要 hardcode POSIX-only 假设
  - CLI 测试用 `typer.testing.CliRunner`，不要依赖真实交互 TTY
  - REPL 测试通过 `input="/exit\n"` 驱动退出

### 5. 强制执行顺序（TDD，不可跳步）

严格按以下顺序，禁止“先写实现再补测试”：

#### Step A — 项目元数据

1. 写入 `pyproject.toml`（name/version/scripts/dependency-groups/ruff/pytest 与计划一致）
2. 写入 `.gitignore`（含 `.venv/`, `__pycache__/`, `.pytest_cache/`, `.ruff_cache/`, `.miku/`, dist/build/egg-info）
3. 写入最小 `README.md`

#### Step B — 失败测试先落地

创建 `tests/test_cli_init.py`，至少包含计划中的两个用例：

1. `test_init_creates_miku_project_files`
   - `init --project-root <tmp>` exit 0
   - 存在 `.miku/`、`project.md`、`config.toml`、`sessions/`
   - stdout 含 `Initialized MikuCode`
2. `test_chat_command_starts_and_exits`
   - `chat --project-root <tmp>` + input `/exit`
   - exit 0 且 stdout 含 `MikuCode`

#### Step C — 红灯确认

```bash
uv sync
uv run pytest tests/test_cli_init.py -v
```

期望：FAIL（模块不存在或命令未实现）。若意外 PASS，说明范围/测试错误，必须停下诊断。

#### Step D — 最小实现（只够让测试与接口契约通过）

实现：

1. `src/mikucode/__init__.py` → `__version__ = "0.1.0"`
2. `config.py`：
   - `ensure_miku_dir`：创建 `sessions/`、`patches/`、`backups/`；若不存在则写默认 `project.md` 与 `config.toml`（内容与计划示例一致）
   - 已有文件 **不要覆盖**（幂等 init）
   - `load_config`：resolve root → ensure → 返回 `MikuConfig`
3. `cli/repl.py`：
   - `MikuRepl` 启动时 `load_config`
   - 打印带 “MikuCode” 的标题
   - 支持 `/exit` | `exit` | `quit` | EOF 退出
   - 其他输入：占位提示 “Runtime is not connected yet...”
4. `cli/main.py`：
   - `init` / `chat` / callback one-shot 占位
   - `init` 成功文案需让测试断言命中（含 `Initialized MikuCode`）

#### Step E — 绿灯 + 静态检查

```bash
uv run pytest tests/test_cli_init.py -v
uv run ruff check src tests
```

两者都必须 PASS。

#### Step F — 可选 smoke（推荐）

```bash
uv run miku --help
uv run miku init --project-root <某个临时目录>
```

确认 help 与 init 可用。不要在仓库根目录留下可提交的 `.miku/`（应被 gitignore）。

#### Step G — 提交（仅在用户/流程允许时）

### 7. 设计与代码质量闸门

- **边界清晰**：`cli/` 不写 agent 逻辑；配置与目录初始化集中在 `config.py`
- **幂等**：`ensure_miku_dir` 可重复调用；mkdir 用 `exist_ok=True`；文件只在缺失时创建
- **路径安全（本 Task 最小）**：始终 `project_root.resolve()`；后续 Task 会加 sandbox，此处至少避免相对路径混乱
- **类型与现代 Python**：使用 `Path`、`dataclass(frozen=True)`、`str | None` 等 3.11+ 语法
- **测试隔离**：只用 `tmp_path`，不污染真实仓库
- **错误处理**：本 Task 不要求完善 CLI 错误 UX；但 init/chat 在正常路径下 exit 0
- **不要**为了“更完美”重写计划中的 API 形状；后续 Task 依赖这些名字与签名

### 8. 已知坑与对抗式检查（必须自检）

执行中主动检查并规避：

1. **Typer 参数名**：测试使用 `--project-root`；Option 参数名必须匹配（`project_root`）
2. **stdout 文案**：测试断言 `Initialized MikuCode` 与 `MikuCode`；文案漂移会导致假失败
3. **chat + input**：`CliRunner.invoke(..., input="/exit\n")`；REPL 必须读 stdin 而非假定交互终端
4. **package discoverability**：缺少 `__init__.py` 或 scripts 配置错误会导致 import/entry fail
5. **重复 init 覆盖用户编辑**：禁止 overwrite 已有 `project.md`/`config.toml`
6. **把 `.miku/` 提交进 git**：gitignore 必须生效；不要 `git add .miku`
7. **超范围实现 runtime stub**：禁止创建空的 `runtime/` 等“提前铺路”目录，除非计划要求
8. **Windows 行尾 / 编码**：写文件统一 `encoding="utf-8"`

若测试失败：先读失败断言与 traceback，修最小集合，禁止用删测试/改断言方式“过关”，除非断言与计划冲突（冲突时以计划为准并在总结中说明）。

## Design notes (meta)

Why this goal is written this way:

1. **Promote plan steps into DoD + prohibitions** — agents tend to pre-build runtime; Out of Scope blocks that.
2. **Red-light gate is mandatory** — forces real TDD, not post-hoc tests.
3. **API contract is frozen** — Task 2+ imports these symbols; signature drift is expensive.
4. **Windows + uv are first-class constraints** — match the real workspace.
5. **Acceptance is machine-checkable** — commands and file presence, not “looks done.”
