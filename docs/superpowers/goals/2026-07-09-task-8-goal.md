## Executable `/goal` body

实现 MikuCode B 版 Task 8：OpenAI-Compatible Provider、REPL Runtime Wiring、One-Shot Mode 和 Undo/Diff Commands。严格按 docs/superpowers/plans/2026-07-07-mikucode-implementation.md 的 Task 8 执行，不得提前实现 Task 9+ 功能。

### 1. 目标（What / Why）

在仓库 A:\study\AI\LLM\Fable-5 中，将 MikuCode 从"各模块独立可测"升级为"CLI 真正连接 runtime"的端到端可用系统：

- `OpenAICompatibleProvider`：通过 httpx 调用 OpenAI-compatible chat completions API，支持 base_url/api_key/model 配置
- `build_registry(project_root)`：组装完整 ToolRegistry（filesystem + search + shell + testing）
- `build_provider()`：根据环境变量选择 MockProvider 或 OpenAICompatibleProvider
- CLI one-shot 模式：`miku "task"` 真正启动 AgentRuntime 执行任务
- CLI REPL 模式：`miku chat` 中用户输入真正调用 runtime
- CLI undo 命令：`miku undo` 调用 UndoManager
- 环境变量：`MIKU_OPENAI_BASE_URL`、`MIKU_OPENAI_API_KEY`、`MIKU_MODEL`、`MIKU_MOCK_RESPONSES`

本 Task 是 MikuCode 从"可测试模块集合"变为"可用产品"的关键转折：CLI 现在真正连接 runtime，用户可以通过 `miku chat` 或 `miku "task"` 让 agent 搜索文件、读取内容、提出 patch、运行测试。

### 7. 设计与代码质量闸门

- **CLI 架构不变**：保持单命令手动 dispatch，不引入 Typer group。`app = typer.Typer()` + `@app.command()` + `@app.callback(invoke_without_command=True)` 的模式不变
- **Mock 优先测试**：所有 CLI 测试通过 `MIKU_MOCK_RESPONSES` 环境变量使用 MockProvider，不依赖真实 API
- **环境变量驱动**：`build_provider()` 完全由环境变量决定使用哪个 provider，无硬编码 API key
- **循环导入防护**：若 `main.py` 和 `repl.py` 互相导入 `build_registry`/`build_provider`，提取到 `cli/factory.py`
- **REPL 每轮新建 runtime**：每次用户输入创建新的 AgentRuntime，不跨轮次共享状态（B 版本简化）
- **类型与现代 Python**：使用 `Path`、`str | None` 等 3.11+ 语法
- **测试隔离**：只用 `tmp_path` 和 `monkeypatch`，不污染真实仓库和环境
- **不要**为了"更完美"重写计划中的 API 形状；后续 Task 依赖这些名字与签名

### 8. 已知坑与对抗式检查（必须自检）

执行中主动检查并规避：

1. **循环导入**：`cli/main.py` 导入 `cli/repl.py`（MikuRepl），`cli/repl.py` 导入 `cli/main.py`（build_provider, build_registry）。这是最可能出问题的地方。解决方案：提取到 `cli/factory.py`
2. **Typer callback 与 command 的交互**：`@app.callback(invoke_without_command=True)` 的 `main` 函数在有子命令时不应执行。one-shot task 通过 `typer.Argument` 传入，需确保 `miku chat` 和 `miku undo` 不触发 one-shot 逻辑
3. **MIKU_MOCK_RESPONSES 的 JSON 解析**：环境变量值是 JSON 数组字符串，需 `json.loads` 后逐项转为字符串。数组元素可能是 dict（需 `json.dumps`）或已是字符串
4. **httpx 同步调用**：`OpenAICompatibleProvider.complete` 使用 `httpx.post`（同步），在 REPL 中可能阻塞。B 版本可接受，C 版本考虑异步
5. **API key 缺失时的错误处理**：`OpenAICompatibleProvider.complete` 在无 api_key 时 raise RuntimeError，CLI 层需捕获并友好提示
6. **undo 命令的 exit_code**：即使无备份（undo 失败），CLI 也应 exit 0，只在 stdout 显示 "No backup"
7. **REPL 中 /exit 仍需工作**：修改 repl.py 后，`/exit`、`exit`、`quit` 退出逻辑不能被破坏
8. **monkeypatch 环境变量**：测试中用 `monkeypatch.setenv` 设置 `MIKU_MOCK_RESPONSES`，需确保测试后环境恢复

若测试失败：先读失败断言与 traceback，修最小集合，禁止用删测试/改断言方式"过关"，除非断言与计划冲突（冲突时以计划为准并在总结中说明）。
