- **Trace replay 只读**：`render_trace` 只读取和格式化，不修改 trace 文件

- **Benchmark fixture 独立**：`benchmarks/smoke/parser_bug/` 是独立的小项目，不依赖 mikucode 源码
- **README 完整但不膨胀**：覆盖所有关键信息，但不写 architecture deep-dive 或 API reference（留给 C 版本文档站）
- **CLI 子命令模式**：`trace show` 和 `bench smoke` 通过 `add_typer` 添加，这是 Typer 标准子命令模式，不违反单命令 dispatch 原则
- **类型与现代 Python**：使用 `Path`、`str | None` 等 3.11+ 语法
- **测试隔离**：只用 `tmp_path` 和 `monkeypatch`，不污染真实仓库和环境
- **不要**为了"更完美"重写计划中的 API 形状

### 8. 已知坑与对抗式检查（必须自检）

执行中主动检查并规避：

1. **add_typer 与 callback 的交互**：`app.add_typer(trace_app, name="trace")` 可能与 `@app.callback(invoke_without_command=True)` 冲突。需确保 `miku` 无参时仍显示 help，`miku trace show` 和 `miku bench smoke` 正常工作
2. **smoke benchmark 的临时目录**：`run_smoke_benchmark` 在 `project_root` 下创建 `bench-smoke-repo/`，需确保不污染用户项目。使用 `tmp_path` 或在 `.gitignore` 中添加 `bench-smoke-repo/`
3. **trace 文件路径参数**：`miku trace show <path>` 的 path 参数需正确解析为 `Path` 对象，Windows 路径可能含空格
4. **README 替换而非追加**：必须替换当前最小版 README，不是在后面追加内容
5. **benchmark fixture 的 Python 导入**：`test_parser_demo.py` 中 `from parser_demo import parse_value` 需要正确的 Python path 配置。fixture 的 `pyproject.toml` 需配置 `pythonpath`
6. **全量测试回归**：本 Task 修改 `cli/main.py`，可能影响 Task 1/8 的测试。必须运行 `uv run pytest -v` 全量测试
7. **bench smoke 的 MockProvider**：`run_smoke_benchmark` 调用 `build_provider()`，需确保 `MIKU_MOCK_RESPONSES` 环境变量在测试中正确设置
8. **render_trace 的空行处理**：JSONL 文件可能含空行，`render_trace` 需跳过空行

若测试失败：先读失败断言与 traceback，修最小集合，禁止用删测试/改断言方式"过关"，除非断言与计划冲突（冲突时以计划为准并在总结中说明）。
