## Executable `/goal` body

### 1. 目标（What / Why）

在仓库 A:\study\AI\LLM\Fable-5 中，基于 Task 2 已落地的 `ToolResult`，建立 MikuCode 的工具执行底座：

- 工具注册表（`ToolRegistry`）：统一的工具注册、查询、执行入口，unknown tool 返回 `ok=False` 的 `ToolResult`
- 工具定义（`ToolDefinition`）：每个工具自带 schema / 风险等级 / 权限 / 超时 / 输出上限
- 路径沙箱（`PathPolicy`）：所有文件系统工具的路径必须经它 canonicalize + 边界检查，拒绝逃逸、symlink escape、secret 文件、`.git` 内部
- 文件系统工具：`read_file`（字符截断 + 二进制拒绝）、`list_files`（忽略生成/缓存目录）
- 搜索工具：`search_text`（行级匹配 + `max_results` 上限）

本 Task 是 Task 4（AgentRuntime 调用工具）与 Task 5（PatchEngine 复用 `PathPolicy`）的直接前置：工具契约、路径策略名字与签名必须稳定。

#### Step D - 实现 PathPolicy

`src/mikucode/permissions/policy.py`：

- `PathPolicy.__init__(project_root)`：`self.project_root = project_root.resolve()`
- `ensure_inside_project(path: str) -> Path`：
  1. `candidate = (self.project_root / path).resolve()` —— `.resolve()` 会跟随 symlink，canonicalize 后再判边界，可拦截 symlink escape
  2. 边界检查：`candidate == self.project_root` 或 `self.project_root in candidate.parents`，否则抛 `PermissionError("... outside project root ...")`
  3. secret 拒绝：`candidate.name` 命中 `{".env", "id_rsa", "id_ed25519", "credentials.json"}` 或后缀 `{".pem", ".key"}`，且不在允许集 `{".env.example", ".env.sample"}`，抛 `PermissionError("... secret-like ...")`（信息含 `secret`）
  4. `.git` 内部拒绝：`".git" in candidate.parts` 且 `candidate.name != ".gitignore"`，抛 `PermissionError("... .git internals ...")`
  5. 返回 `candidate`

> 实现细节可参考 plan Step 4 示例，但边界判定必须用 `.resolve()` 后的 canonical path（非 `os.path.realpath` 的字符串拼接），确保 symlink 被解析后再比较。注意 Windows 上 `resolve()` 对不存在路径也能正常工作。

#### Step E - 实现 filesystem 工具

`src/mikucode/tools/filesystem.py`：

- 常量 `_IGNORED_DIRS = {".git", ".venv", "node_modules", "dist", "build", "__pycache__", ".pytest_cache"}`
- `register_filesystem_tools(registry, project_root)` 注册：
  - `read_file`：
    - 经 `PathPolicy.ensure_inside_project` 解析
    - 读时先 `bytes = resolved.read_bytes()`，检测是否含 `b"\x00"`（null byte）判断二进制；若是二进制，返回 `ok=False`（建议 summary 含 `binary`）。或用 `read_text` 捕获 `UnicodeDecodeError` 降级为 `ok=False`
    - 文本成功后截断至 `output_limit`（默认 20000）字符，`metadata["truncated"]` 反映是否截断
    - 任何异常（权限拒绝、文件不存在、解码失败）都返回 `ToolResult(ok=False, tool="read_file", ...)`，**不抛出**
    - `metadata` 至少含 `path`
  - `list_files`：
    - `project_root.rglob("*")` 遍历，跳过任何 part 命中 `_IGNORED_DIRS` 的项
    - 只收集 `is_file()`，相对路径用 `as_posix()`
    - 结果排序，content 用 `\n` 连接；summary 含文件数

#### Step F - 实现 search 工具

`src/mikucode/tools/search.py`：

- 复用同一 `_IGNORED_DIRS` 集合
- `register_search_tools(registry, project_root)` 注册 `search_text`：
  - 参数：`query`（必填）、`max_results`（默认 50，可由参数覆盖）
  - 遍历项目内非忽略目录的文件，`read_text(encoding="utf-8")`；捕获 `UnicodeDecodeError` 跳过（天然跳过二进制文件）
  - 行级匹配 `query in line`，命中格式 `"<rel posix>:<lineno>: <stripped line>"`
  - 命中数达到 `max_results` 立即返回（early exit，避免大仓库全量扫描）
  - search_text 是否经 PathPolicy：search 遍历整个项目根，本身不接用户 path 参数，因此**不**对遍历结果逐个调用 `ensure_inside_project`（避免性能问题），但必须保证只读、只遍历 `_IGNORED_DIRS` 之外的路径，且不跟随 symlink（用 `rglob` 默认行为即可；若需更严，可加 `item.is_symlink()` 跳过）

### 7. 设计与代码质量闸门

- **边界清晰**：`tools/` 只做工具实现与注册；路径安全决策集中在 `permissions/policy.py`；工具不直接 `open` 任意路径，必须经 `PathPolicy`
- **fail-soft**：工具 executor 永远返回 `ToolResult`，不向调用方抛异常；权限拒绝、文件缺失、解码失败都转化为 `ok=False`
- **幂等与可重复**：`register_*` 可被多次调用（对同一 registry 重复注册同名工具应抛 `ValueError`，这是显式契约，不是 bug）
- **类型与现代 Python**：`Path`、`Callable[[dict[str, Any]], ToolResult]`、`dict[str, Any]`、`str | None` 等 3.11+ 语法；pydantic v2 BaseModel
- **输出上限**：所有可能产出大文本的工具（`read_file`、`search_text`）都必须有上限（20000 / max_results），防止单次工具调用撑爆 context
- **测试隔离**：只用 `tmp_path`，不污染真实仓库；不在仓库根目录遗留测试产物
- **路径安全是本 Task 核心**：宁可误拒（返回 `ok=False`）也不可放行逃逸/secret/.git 路径
- **不要**为了“更完美”改动 `ToolResult` 的字段（Task 2 已冻结 `ok/tool/summary/content/metadata`）；不要重写计划中的 API 形状，后续 Task 依赖这些名字

### 8. 已知坑与对抗式检查（必须自检）

执行中主动检查并规避，对每条至少自问“这会失败吗”：

1. **path traversal**：`../secret.txt`、绝对路径 `C:\...`、`..%2f` 之类。必须 `.resolve()` 后用 parents 判定边界；**不能**只做字符串前缀检查（`startswith` 会被 `project_root_evil/` 之类前缀欺骗）
2. **symlink escape**：项目内放一个指向项目外的 symlink，`read_file` 跟随它会读到外部文件。`.resolve()` 会展开 symlink 到真实路径，随后边界检查会拒绝。自检：在 `tmp_path` 内建 symlink 指向 `tmp_path` 外文件，断言被拒（Windows 创建 symlink 可能需权限，可跳过实际测试但代码路径必须正确）
3. **secret file denial**：`.env`、`.env.local`、`id_rsa`、`server.pem`、`private.key`、`credentials.json` 必须拒。注意允许 `.env.example` / `.env.sample`（白名单优先于后缀匹配）
4. **binary file read**：`read_file` 读 `.png`/编译产物会 `UnicodeDecodeError` 或乱码。必须捕获并返回 `ok=False`，不能把乱码塞进 content。推荐 `read_bytes()` 后查 `b"\x00"` 提前判定
5. **忽略目录漏判**：`list_files` / `search_text` 必须忽略 `.git/.venv/node_modules/dist/build/__pycache__/.pytest_cache`。注意判定方式是“路径任一 part 命中集合”，不是只看顶层目录名（否则 `src/.venv/x` 会被漏放）
6. **read_file 截断边界**：恰好 20000 字符不截断，20001 截断；`metadata["truncated"]` 用 `len(content) > output_limit`，截断用 `[:output_limit]`
7. **search_text 大仓库**：无 `max_results` 上限会全量扫描 + 拼巨型 content。必须 early exit；`max_results` 来自参数，默认 50
8. **ToolRegistry unknown tool**：`execute("不存在", {})` 必须返回 `ok=False`，**不可** `KeyError`。Task 4 的 AgentRuntime 会直接把模型输出的 tool 名丢进来
9. **Windows rglob 顺序**：`rglob` 顺序不保证稳定；`list_files` 必须 `sort()` 后再输出，否则测试断言不稳定
10. **相对路径 posix 化**：Windows 上 `relative_to` 返回反斜杠路径；测试断言用 `src/main.py`，必须 `as_posix()`
11. **重复注册**：同一 registry 注册两次 `read_file` 应抛 `ValueError`（显式失败优于静默覆盖）
12. **.gitignore 例外**：`.git` 内部拒绝时，`.gitignore` 应放行（项目根的 `.gitignore` 是合法文件）。自检边界条件别把 `.gitignore` 一起拒了
13. **PermissionError 文案**：测试断言 `outside project root` / `secret` 子串；文案漂移会导致假失败。secret 拒绝信息必须含 `secret` 字样
14. **PathPolicy 不读文件内容**：`ensure_inside_project` 只做路径决策，不 IO；secret 判定基于文件**名**而非内容（内容扫描是 TraceRecorder redaction 的职责，本 Task 不做）

若测试失败：先读失败断言与 traceback，修最小集合，禁止用删测试/改断言方式“过关”，除非断言与计划冲突（冲突时以计划与本 goal 契约为准并在总结中说明）。

## Design notes (meta)

Why this goal is written this way:

1. **Security is the load-bearing requirement** - PathPolicy is the only thing standing between the model and arbitrary file reads; the adversarial checklist forces explicit handling of traversal, symlink escape, secrets, and .git.
2. **Red-light gate is mandatory** - forces real TDD on the path sandbox before any executor is written.
3. **API contract is frozen** - Task 4 (AgentRuntime.execute) and Task 5 (PatchEngine reuses PathPolicy) import these symbols; signature drift is expensive. Field names (`timeout`/`output_limit`) are pinned here over the plan's example.
4. **Fail-soft executors** - tools must never raise to the runtime; the unknown-tool path returning `ok=False` (not KeyError) is a hard contract because Task 4 feeds raw model tool names in.
5. **Output limits are first-class** - `read_file` (20000) and `search_text` (max_results) prevent a single tool call from saturating context; this is enforced in DoD, not just code.
6. **Windows + uv are first-class constraints** - `as_posix()` for assertions, `resolve()` for canonicalization, `sort()` for stable listing.
7. **Acceptance is machine-checkable** - commands, file presence, and test assertions, not "looks done."
