## Executable `/goal` body

实现 MikuCode B 版 Task 7：Context Builder 和 Project/Session Memory。严格按 docs/superpowers/plans/2026-07-07-mikucode-implementation.md 的 Task 7 执行，不得提前实现 Task 8+ 功能。

### 1. 目标（What / Why）

在仓库 A:\study\AI\LLM\Fable-5 中，为 MikuCode 添加上下文构建和项目/会话记忆能力：

- `build_file_tree(project_root) -> str`：生成项目文件树文本，忽略 `.git`、`.miku`、`.venv` 等生成目录，最多 200 个文件
- `ContextBuilder.build(state) -> list[dict]`：组装完整上下文消息列表，包含 task、project_memory、file_tree、recent_observations、verification_state
- `MemoryStore.read_project_memory() -> str`：读取 `.miku/project.md` 内容
- `MemoryStore.add_project_memory(text, source) -> None`：向 `project.md` 追加带时间戳和来源的记忆条目
- `AgentRuntime._build_messages` 替换为 `ContextBuilder.build`：从硬编码消息格式升级为上下文感知的消息构建

本 Task 是 MikuCode 从"无状态循环"升级为"上下文感知 agent"的关键：模型现在能看到项目结构、历史记忆和最近观察，做出更合理的决策。

### 2. 范围（In Scope）

仅创建/修改以下文件（与计划一致）：

- `src/mikucode/context/file_tree.py`
- `src/mikucode/context/builder.py`
- `src/mikucode/memory/store.py`
- `src/mikucode/runtime/agent.py`（用 `ContextBuilder` 替换 `_build_messages`）
- `tests/test_context_and_memory.py`

接口契约（必须实现且可被后续 Task import）：

- `build_file_tree(project_root: Path, max_files: int = 200) -> str`
- `ContextBuilder(project_root: Path, max_context_chars: int = 60_000)`，方法 `build(state: AgentState) -> list[dict]`
- `MemoryStore(project_root: Path)`，方法 `read_project_memory() -> str`、`add_project_memory(text: str, source: str) -> None`
- `AgentRuntime._build_messages` 委托给 `ContextBuilder.build`

  - `uv sync`

  - `uv run ruff check ...`

### 7. 设计与代码质量闸门

- **字符预算硬限制**：`max_context_chars = 60_000`，超出截断并追加 `[context truncated]`。这是防止上下文爆炸的核心机制
- **观察窗口**：`recent_observations` 只取最近 5 个，避免历史膨胀
- **文件树限制**：`max_files = 200`，大型项目不生成完整树
- **记忆追加模式**：`add_project_memory` 只追加不覆盖，保护用户已有记忆
- **忽略 .miku 目录**：file_tree 忽略 `.miku/`，避免将内部状态文件暴露给模型
- **类型与现代 Python**：使用 `Path`、`str | None` 等 3.11+ 语法
- **测试隔离**：只用 `tmp_path`，不污染真实仓库
- **不要**为了"更完美"重写计划中的 API 形状；后续 Task 依赖这些名字与签名

### 8. 已知坑与对抗式检查（必须自检）

执行中主动检查并规避：

1. **ContextBuilder 与 Task 4 测试兼容**：`_build_messages` 的返回格式从硬编码三条消息变为 ContextBuilder 动态生成。Task 4 的 MockProvider 依赖消息格式，需确保 MockProvider 不依赖消息内容（它只返回预设响应，所以应该兼容）
2. **project.md 初始内容**：Task 1 的 `ensure_miku_dir` 已创建默认 `project.md`。`read_project_memory` 应能读取这个默认内容
3. **add_project_memory 的追加格式**：追加的 `## Memory entry` 块需与默认 `project.md` 的 Markdown 格式兼容
4. **JSON 序列化中的特殊字符**：project_memory 和 file_tree 可能含 JSON 特殊字符（引号、反斜杠），`json.dumps(ensure_ascii=False)` 应正确处理
5. **截断位置**：字符预算截断在 JSON 字符串中间，可能导致模型收到不完整 JSON。B 版本可接受此限制，截断标记 `[context truncated]` 至少告知模型
6. **file_tree 的 rglob 性能**：大型项目中 `rglob("*")` 可能慢。`max_files=200` 限制结果数量，但遍历本身仍可能慢。B 版本可接受
7. **MemoryStore 的 ensure_miku_dir 副作用**：`MemoryStore.__init__` 调用 `ensure_miku_dir`，可能在测试中创建意外目录。使用 `tmp_path` 隔离
8. **context/builder.py 的循环导入**：`ContextBuilder` 导入 `MemoryStore` 和 `file_tree`，`AgentRuntime` 导入 `ContextBuilder`。需确保无循环依赖

若测试失败：先读失败断言与 traceback，修最小集合，禁止用删测试/改断言方式"过关"，除非断言与计划冲突（冲突时以计划为准并在总结中说明）。
