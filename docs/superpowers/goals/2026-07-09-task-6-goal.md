## Executable `/goal` body

实现 MikuCode B 版 Task 6：Shell Risk Classifier、Shell Tool 和 Verification Engine。严格按 docs/superpowers/plans/2026-07-07-mikucode-implementation.md 的 Task 6 执行，不得提前实现 Task 7+ 功能。

### 1. 目标（What / Why）

在仓库 A:\study\AI\LLM\Fable-5 中，为 MikuCode 添加命令风险分类、受控 shell 执行和测试验证能力：

- `classify_command(command) -> CommandRisk`：将 shell 命令分为 low/medium/critical 三级，返回 level、decision（allow/ask/deny）和 reasons
- `register_shell_tool`：注册 `run_shell` 工具到 ToolRegistry，只自动执行 low risk 命令，medium 返回 ask 提示，critical 拒绝
- `detect_test_command(project_root)`：检测项目测试命令，优先 `uv run pytest`
- `VerificationRunner.run() -> ToolResult`：通过 shell tool 执行测试命令，返回 exit_code 和输出
- `register_testing_tools`：注册 `detect_tests` 和 `run_tests` 工具
- `summarize_verification`：将 ToolResult 转为结构化验证摘要

本 Task 是 MikuCode 安全模型的关键层：所有 shell 执行必须经过风险分类，测试验证是 agent 决策的证据基础。

### 7. 设计与代码质量闸门

- **默认拒绝**：未分类命令归为 medium/ask，不自动执行。宁可多问，不可误放
- **三级决策**：low=allow（自动执行）、medium=ask（返回提示，B 版本不弹 UI）、critical=deny（硬拒绝）
- **超时保护**：`subprocess.run` 必须设 timeout，默认 30 秒，防止命令挂起
- **输出截断**：shell 输出截断 20_000 字符，防止内存爆炸
- **VerificationRunner 独立**：不依赖全局 ToolRegistry，每次 run 创建临时 registry
- **测试命令检测优先 uv**：`detect_test_command` 优先 `uv run pytest`，与项目 uv 工作流一致
- **类型与现代 Python**：使用 `Path`、`dataclass(frozen=True)`、`str | None` 等 3.11+ 语法
- **测试隔离**：只用 `tmp_path`，不污染真实仓库
- **不要**为了"更完美"重写计划中的 API 形状；后续 Task 依赖这些名字与签名

### 8. 已知坑与对抗式检查（必须自检）

执行中主动检查并规避：

1. **Windows subprocess**：`subprocess.run(shell=True)` 在 Windows 上使用 cmd.exe；`python -c "print('hello')"` 的引号在 cmd 中可能需要调整。测试中优先使用 `python -c "print('hello')"` 格式
2. **critical 模式误判**：`curl` 本身不是 critical，只有 `curl | bash` 才是。分类器需精确匹配管道组合，不要把 `curl https://api.example.com/data`（合法 API 调用）误判为 critical
3. **medium 命令的 ask 语义**：B 版本中 `ask` 意味着"返回提示，不执行"，不是"弹确认框"。测试断言 `result.ok is False`
4. **VerificationRunner 的 ToolResult.tool**：runner 将 tool 从 `"run_shell"` 改为 `"run_tests"`，确保调用方能区分
5. **detect_test_command 的默认值**：即使没有 `pyproject.toml`，默认也返回 `"uv run pytest"`，因为 MikuCode 项目本身就是 uv 管理的
6. **subprocess 超时异常**：`subprocess.TimeoutExpired` 需要捕获并转为 ToolResult，不能让异常泄漏
7. **shell 注入**：B 版本不做完整 shell 注入防护（C 版本职责），但 risk classifier 至少拦截 `| bash`、`| sh`、`sudo` 等明显危险模式
8. **testing.py 的循环导入**：`tools/testing.py` 导入 `verification/detector.py` 和 `verification/runner.py`，而 runner 导入 `tools/shell.py`。需确保无循环依赖

若测试失败：先读失败断言与 traceback，修最小集合，禁止用删测试/改断言方式"过关"，除非断言与计划冲突（冲突时以计划为准并在总结中说明）。
